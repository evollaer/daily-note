# 新手教程：在 Spring Boot 项目中新增一个登录功能

> 本文以"新增访客登录"为例，手把手教你在这个项目里添加一套完整的登录模块。
> 需要的基础：了解 Java、MySQL、Spring Boot 基本概念。

---

## 一、先搞清楚：登录功能涉及哪些东西？

在这个项目里，一个完整的登录功能需要 **6 个部分**配合工作：

```
数据库表  →  实体类  →  Mapper  →  Service  →  Controller  →  拦截器配置
   ↑                                                    ↓
   └──────────── 用户请求进来，返回 Token ────────────────┘
```

你可以把这个流程理解为：
1. **数据库表**：存账号密码的地方
2. **实体类（POJO）**：Java 代码里用来"装"数据库数据的容器
3. **Mapper**：负责去数据库查数据的"快递员"
4. **Service**：处理业务逻辑的"大脑"（比如验证密码对不对）
5. **Controller**：对外提供 HTTP 接口的"前台"（前端调用的入口）
6. **拦截器配置**：告诉系统"登录接口不用验 Token，其他接口都要验"

下面一步一步来。

---

## 二、第 1 步：创建数据库表

### 2.1 为什么要先建表？

用户的账号密码必须存到数据库里，登录时才能查出来比对。

### 2.2 在哪建表？

打开 MySQL Workbench，在 `bms_boot` 数据库里执行：

```sql
CREATE TABLE `t_guests` (
  `guest_id` bigint(11) NOT NULL AUTO_INCREMENT COMMENT '访客ID',
  `username` varchar(32) NOT NULL COMMENT '用户名',
  `password` varchar(32) NOT NULL COMMENT '密码(MD5加密)',
  `guest_name` varchar(10) NOT NULL COMMENT '访客姓名',
  `status` int(1) NOT NULL COMMENT '1表示可用 0表示禁用',
  `create_time` datetime NOT NULL COMMENT '创建时间',
  `update_time` datetime NOT NULL ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`guest_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8;
```

### 2.3 字段说明

| 字段名 | 类型 | 说明 |
|---|---|---|
| `guest_id` | bigint | 主键，自增，唯一标识这个访客 |
| `username` | varchar | 登录用的用户名 |
| `password` | varchar | 登录密码（存的是加密后的，不是明文！） |
| `guest_name` | varchar | 访客的真实姓名 |
| `status` | int | 1=正常可用，0=被禁用，不能登录 |
| `create_time` | datetime | 账号创建时间 |
| `update_time` | datetime | 最后更新时间 |

### 2.4 验证建表成功

```sql
USE bms_boot;
SHOW TABLES;        -- 应该能看到 t_guests
DESC t_guests;      -- 查看表结构详情
```

## 三、第 2 步：写实体类（POJO）

### 3.1 什么是实体类？

实体类就是一个普通的 Java 类，**每个属性对应数据库表的一列**。它的作用是：把数据库里的一行数据"装"进 Java 对象里，方便代码操作。

### 3.2 在哪写？

新建文件：`src/main/java/com/book/backend/pojo/Guests.java`

```java
package com.book.backend.pojo;

import com.baomidou.mybatisplus.annotation.*;
import lombok.Data;
import java.io.Serializable;

/**
 * 访客实体类，对应数据库表 t_guests
 */
@Data                           // Lombok 注解，自动生成 getter、setter、toString
@TableName(value = "t_guests")   // 告诉 MyBatis-Plus：这个类对应哪张表
public class Guests implements Serializable {
    
    @TableId(type = IdType.AUTO)  // 主键，自增
    private Long guestId;
    
    private String username;       // 用户名
    private String password;       // 密码
    private String guestName;      // 访客姓名
    private Integer status;        // 状态 1可用 0禁用
    
    @TableField(fill = FieldFill.INSERT)         // 插入时自动填充
    private String createTime;
    
    @TableField(fill = FieldFill.INSERT_UPDATE)  // 插入和更新时自动填充
    private String updateTime;
}
```

### 3.3 关键注解解释

| 注解 | 作用 |
|---|---|
| `@Data` | Lombok 提供，自动生成 getter、setter、toString、equals、hashCode |
| `@TableName("t_guests")` | 指定这个类映射的数据库表名 |
| `@TableId(type = IdType.AUTO)` | 标记这是主键，自增策略 |
| `@TableField(fill = ...)` | 自动填充字段，createTime 在插入时填，updateTime 在插入和更新时填 |

## 四、第 3 步：写 Mapper

### 4.1 什么是 Mapper？

Mapper 是**数据访问层**，负责和数据库打交道。在这个项目里，我们用 MyBatis-Plus，它已经把常用的增删改查都封装好了，我们只需要**继承一个接口**就行。

### 4.2 在哪写？

新建文件：`src/main/java/com/book/backend/mapper/GuestsMapper.java`

```java
package com.book.backend.mapper;

import com.baomidou.mybatisplus.core.mapper.BaseMapper;
import com.book.backend.pojo.Guests;

/**
 * 访客数据访问层
 * 继承 BaseMapper 后，自动拥有 insert、delete、update、select 等方法
 */
public interface GuestsMapper extends BaseMapper<Guests> {
    // 暂时不需要写自定义方法
}
```

### 4.3 为什么这么简单？

因为 `BaseMapper<Guests>` 已经提供了这些方法：
- `insert(Guests entity)` — 插入
- `deleteById(Long id)` — 按 ID 删除
- `updateById(Guests entity)` — 按 ID 更新
- `selectById(Long id)` — 按 ID 查询
- `selectList(...)` — 条件查询列表

如果需要写复杂 SQL（比如多表联查），才需要在 `src/main/resources/mapper/` 下写 XML 文件。

## 五、第 4 步：写 Service（接口 + 实现类）

### 5.1 为什么要分接口和实现类？

这是 Java 的**面向接口编程**思想：
- **接口（Service）**：定义"要做什么"（比如定义 login 方法）
- **实现类（ServiceImpl）**：具体"怎么做"（比如怎么查库、怎么验密码）

好处是：如果以后换一套实现逻辑，只需要换实现类，其他地方不用改。

### 5.2 Service 接口

新建文件：`src/main/java/com/book/backend/service/GuestsService.java`

```java
package com.book.backend.service;

import com.baomidou.mybatisplus.extension.service.IService;
import com.book.backend.common.R;
import com.book.backend.pojo.Guests;

/**
 * 访客服务接口
 * 继承 IService 后，自动拥有大量常用数据库操作方法
 */
public interface GuestsService extends IService<Guests> {
    
    /**
     * 访客登录
     * @param guests 包含用户名和密码
     * @return 统一响应结果 R，成功时 map 里有 token
     */
    R login(Guests guests);
}
```

### 5.3 ServiceImpl 实现类（⭐ 核心逻辑在这里）

新建文件：`src/main/java/com/book/backend/service/impl/GuestsServiceImpl.java`

```java
package com.book.backend.service.impl;

import com.baomidou.mybatisplus.core.conditions.query.LambdaQueryWrapper;
import com.baomidou.mybatisplus.core.toolkit.StringUtils;
import com.baomidou.mybatisplus.extension.service.impl.ServiceImpl;
import com.book.backend.common.R;
import com.book.backend.constant.Constant;
import com.book.backend.mapper.GuestsMapper;
import com.book.backend.pojo.Guests;
import com.book.backend.service.GuestsService;
import com.book.backend.utils.JwtKit;
import org.springframework.stereotype.Service;
import org.springframework.util.DigestUtils;

import javax.annotation.Resource;

@Service  // 标记为 Spring 的 Service 组件，会被自动注入
public class GuestsServiceImpl extends ServiceImpl<GuestsMapper, Guests> 
        implements GuestsService {
    
    // 盐值，和密码拼接后再 MD5，增加安全性
    private static final String SALT = "xiaobaitiao";
    
    @Resource  // 注入 JWT 工具类，用来生成 Token
    private JwtKit jwtKit;

    /**
     * 登录的核心逻辑
     */
    @Override
    public R login(Guests guests) {
        
        // ========== 第 1 步：判空 ==========
        // 如果前端没传用户名，直接返回错误
        if (StringUtils.isBlank(guests.getUsername())) {
            return R.error("用户名不存在");
        }
        
        // ========== 第 2 步：查数据库 ==========
        // LambdaQueryWrapper 是 MyBatis-Plus 的条件构造器
        // 等价 SQL：SELECT * FROM t_guests WHERE username = ?
        LambdaQueryWrapper<Guests> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(Guests::getUsername, guests.getUsername());
        Guests guest = this.getOne(wrapper);  // 查一条记录
        
        // ========== 第 3 步：判断用户是否存在 ==========
        if (guest == null) {
            return R.error("用户名不存在");
        }
        
        // ========== 第 4 步：判断账号是否被禁用 ==========
        // Constant.DISABLE = 0，表示禁用状态
        if (Constant.DISABLE.equals(guest.getStatus())) {
            return R.error("该访客账号已被禁用");
        }
        
        // ========== 第 5 步：验证密码（⭐ 重点）==========
        // 前端传的是明文密码（比如 "123456"）
        // 数据库里存的是：MD5("盐值" + "明文密码")
        // 所以这里也要用同样的方式加密后再比对
        String inputPassword = DigestUtils.md5DigestAsHex(
            (SALT + guests.getPassword()).getBytes()
        );
        
        // 比对加密后的密码和数据库里的密码
        if (!inputPassword.equals(guest.getPassword())) {
            return R.error("用户名或密码错误");
        }
        
        // ========== 第 6 步：密码正确，生成 JWT Token ==========
        // JWT 是一个加密的字符串，里面包含了用户信息，前端以后带着它来证明"我已登录"
        String token = jwtKit.generateToken(guest);
        
        // ========== 第 7 步：组装返回结果 ==========
        R result = new R<>();
        result.setStatus(200);              // HTTP 200 表示成功
        result.setMsg("登录成功");
        result.add("token", token);         // Token 放在 map 里给前端
        result.add("id", guest.getGuestId()); // 用户 ID 也返回
        return result;
    }
}
```

### 5.4 登录流程图解

```
前端传 {username: "guest1", password: "123456"}
           │
           ▼
    ┌──────────────┐
    │  ① 判空      │  username 为空？→ 直接返回错误
    └──────┬───────┘
           ▼
    ┌──────────────┐
    │  ② 查数据库   │  SELECT * FROM t_guests WHERE username = 'guest1'
    └──────┬───────┘
           ▼
    ┌──────────────┐
    │  ③ 判存在    │  查不到？→ "用户名不存在"
    └──────┬───────┘
           ▼
    ┌──────────────┐
    │  ④ 判状态    │  status == 0？→ "账号已被禁用"
    └──────┬───────┘
           ▼
    ┌──────────────┐
    │  ⑤ 验密码    │  MD5("xiaobaitiao123456") == 数据库密码？
    └──────┬───────┘     不一致 → "用户名或密码错误"
           ▼
    ┌──────────────┐
    │  ⑥ 生成 Token │  jwtKit.generateToken(guest)
    └──────┬───────┘
           ▼
    ┌──────────────┐
    │  ⑦ 返回结果   │  {status:200, msg:"登录成功", map:{token:"...", id:1}}
    └──────────────┘
```

## 六、第 5 步：写 Controller

### 6.1 什么是 Controller？

Controller 是**对外暴露 HTTP 接口**的地方。前端通过 URL 调用这里的方法。

### 6.2 在哪写？

新建文件：`src/main/java/com/book/backend/controller/guest/GuestLoginController.java`

```java
package com.book.backend.controller.guest;

import com.book.backend.common.R;
import com.book.backend.pojo.Guests;
import com.book.backend.service.GuestsService;
import io.swagger.annotations.ApiOperation;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

import javax.annotation.Resource;

/**
 * 访客登录控制器
 * @RestController = @Controller + @ResponseBody，表示返回 JSON 而不是页面
 * @RequestMapping("/guest") 表示这个类里所有接口的路径前缀都是 /guest
 */
@RestController
@RequestMapping("/guest")
public class GuestLoginController {
    
    @Resource  // 自动注入 GuestsService
    private GuestsService guestsService;

    /**
     * 访客登录接口
     * @PostMapping("/login") 表示：POST 请求，路径是 /guest/login
     * @RequestBody 表示：把前端传来的 JSON 转成 Guests 对象
     * @ApiOperation 是 Swagger/Knife4j 的注解，用来说明这个接口的作用
     */
    @PostMapping("/login")
    @ApiOperation("访客登录")
    public R login(@RequestBody Guests guests) {
        // 调用 Service 层的登录逻辑
        return guestsService.login(guests);
    }
}
```

### 6.3 注解解释

| 注解 | 作用 |
|---|---|
| `@RestController` | 标记这是 REST 接口类，返回值自动转 JSON |
| `@RequestMapping("/guest")` | 类级别的路径前缀，所有方法的路径前都加 `/guest` |
| `@PostMapping("/login")` | 方法级别的路径，最终 URL 是 `/api/guest/login`（因为项目配置了 context-path: /api） |
| `@RequestBody` | 把 HTTP 请求体（JSON）转成 Java 对象 |
| `@ApiOperation` | Knife4j 文档用的说明文字 |
| `@Resource` | Spring 自动注入（找类型匹配的 Bean 赋值） |

## 七、第 6 步：配置拦截器（确认登录接口放行）

### 7.1 什么是拦截器？

拦截器就像小区门口的保安：
- 大多数请求 → 检查有没有"通行证"（Token）→ 没有就拦住
- 但**登录接口**是"办理通行证"的地方 → 不需要检查，直接放行

### 7.2 需要改吗？

打开 `WebMvcConfig.java`，看这段代码：

```java
@Override
public void addInterceptors(InterceptorRegistry registry) {
    registry.addInterceptor(authInterceptorHandler())
        .excludePathPatterns("/**/login")      // 放行所有以 /login 结尾的接口
        .excludePathPatterns("/doc.html")       // 放行接口文档页面
        .excludePathPatterns("/**/websocket");  // 放行 WebSocket
}
```

因为规则是 `/**/login`，你的 `/guest/login` 已经**自动被放行了**，不需要改！

> 如果你的登录路径不是 `/**/login`（比如 `/guest/doLogin`），才需要手动加一行：`.excludePathPatterns("/guest/doLogin")`

## 九、总结：新增登录功能要改哪些文件

| 步骤 | 文件路径 | 作用 |
|:---:|:---|:---|
| 1 | MySQL 执行 SQL | 创建 `t_guests` 表，存账号密码 |
| 2 | `pojo/Guests.java` | 实体类，映射数据库表 |
| 3 | `mapper/GuestsMapper.java` | 数据访问层，继承 `BaseMapper` |
| 4 | `service/GuestsService.java` | 服务接口，声明 `login()` 方法 |

**最核心的是第 5 步（ServiceImpl）**，前面的都是"模板代码"，后面的都是"接口暴露"。

## 十、常见问题

### Q1：为什么密码要加盐 MD5，不能直接存明文？

**安全！** 如果数据库泄露，明文密码直接暴露。即使黑客拿到 MD5 值，没有盐值也很难反推原始密码。

### Q2：JWT Token 是什么？

简单来说，就是一个**加密的字符串**。用户登录成功后，服务端生成这个字符串给前端，前端每次请求都带上它。服务端收到后解密，就能知道"这是谁、什么时候登录的"。

Token 里存了什么？看 `JwtKit.generateToken()`：
- `username`：用户标识
- `createdate`：签发时间
- `id`：随机编号

### Q3：为什么返回的是 `R<T>` 而不是直接返回对象？

`R<T>` 是项目的**统一响应格式**，所有接口都返回这个结构：
```json
{
  "status": 状态码,
  "msg": "提示信息",
  "data": 实际数据,
  "map": {额外的键值对}
}
```

这样前端处理响应时格式统一，不用猜每个接口返回什么结构。

### Q4：LambdaQueryWrapper 是什么？

MyBatis-Plus 提供的**类型安全查询工具**。对比：

```java
// 传统方式（字符串，写错编译期不报错）
wrapper.eq("username", "guest1");

// Lambda 方式（类型安全，写错直接编译报错）
wrapper.eq(Guests::getUsername, "guest1");
```

项目里**统一用 LambdaQueryWrapper**。
