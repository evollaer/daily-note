### 11. new 操作符原理

1. **创建一个空对象**：new 操作符首先会创建一个空对象，这个对象会继承自构造函数的原型对象。
2. **设置对象的原型链**：将新创建的空对象的原型链指向构造函数的原型对象。
3. **执行构造函数**：调用构造函数，并将新对象作为构造函数的上下文（即this关键字）传入。
4. **返回对象实例**：如果构造函数没有显式返回一个对象，则new操作符会隐式返回这个新对象实例。

### 14. 什么是原型？

**原型链**：原型组成的链，比如函数的原型是Function，Function的原型是Object，Object的原型仍然是Object，一直追溯到最终的原型对象。

- 函数通过 `prototype` 来追溯原型对象
- 对象通过 `__proto__` 来追溯原型对象

**原型链继承示例**：

```javascript
function A(){
  this.name="amy";
}
function B(){
  this.age="20";
}
B.prototype=new A(); // B继承了A,通过原型，形成链条
var a=new B();
console.log(a.name); // 结果：amy
```

### 15. 什么是闭包？

闭包就是一个函数引用另外一个函数的变量，因为变量被引用着所以不会被回收。

**用处**：
1. 可以读取函数内部的变量
2. 让这些变量的值始终保持在内存中，不会在外部函数调用后被自动清除

**特点**：由于局部变量无法共享和长久的保存，而全局变量可能造成变量污染，闭包的出现可以解决长久的保存变量又不会造成全局污染。

### 16. 如何防抖？

**何为防抖**：多次触发事件后，事件处理函数只执行一次，并且是在触发操作结束时执行，一般用于scroll事件。

**解决原理**：对处理函数进行延时操作，若设定的延时到来之前再次触发事件，则清除上一次的延时操作定时器，重新定时。

```javascript
let timer;
window.onscroll = function () {
  if(timer){
    clearTimeout(timer);
  }
  timer = setTimeout(function () {
    let scrollTop = document.body.scrollTop || document.documentElement.scrollTop;
    console.log('滚动条位置：' + scrollTop);
    timer = undefined;
  }, 200);
};
```

### 17. TCP的三次握手和四次挥手

TCP的三次握手和四次挥手实质就是TCP通信的连接和断开。

**三次握手**：为了对每次发送的数据量进行跟踪与协商，确保数据段的发送和接收同步。

**四次挥手**：即终止TCP连接，就是指断开一个TCP连接时，需要客户端和服务端总共发送4个包以确认连接的断开。

**三次握手原理**：
1. 第1次握手：客户端发送一个带有SYN标志的数据包给服务端
2. 第2次握手：服务端接收成功后，回传一个带有SYN/ACK标志的数据包
3. 第3次握手：客户端再回传一个带有ACK标志的数据包，握手结束

**四次挥手原理**：
1. 第1次挥手：客户端发送一个FIN，用来关闭客户端到服务端的数据传送
2. 第2次挥手：服务端收到FIN后，发送一个ACK给客户端
3. 第3次挥手：服务端发送一个FIN，用来关闭服务端到客户端的数据传送
4. 第4次挥手：客户端收到FIN后，发送一个ACK给Server，完成四次挥手

### 18. 事件代理是什么

事件代理是利用事件的冒泡原理来实现的。事件从最深的节点开始，然后逐步向上传播事件。

**示例**：页面上有节点树 `div>ul>li>a`，给最里面的a加一个click点击事件，那么这个事件就会一层一层的往外执行，执行顺序 `a>li>ul>div`。给最外面的div加点击事件，里面的ul、li、a做点击事件时，都会冒泡到最外层的div上，所以都会触发。

### 25. Cookie 和 Token 的区别

**Cookie**：
- 弥补HTTP无状态的问题
- 服务器通过set-cookie令客户端保存cookie
- 下次请求时带上cookie进行身份验证

**Token**：
- 类似cookie的验证信息
- 客户端登录验证后，服务器返回加密token
- 后续请求带上token，服务器直接校验

**Token相对cookie的优势**：
1. 支持跨域访问
2. 无状态化，服务端无需存储token
3. 无需绑定特殊身份验证方案
4. 更适用于移动端
5. 避免CSRF跨站伪造攻击
6. 非常适用于RESTful API

### 27. Promise对象实现Ajax封装

```javascript
const getJSON = function(url, type, data) {
  const promise = new Promise(function(resolve, reject){
    const xmlHttp = new XMLHttpRequest();
    xmlHttp.open(type, url);
    if(type == 'get'){
      xmlHttp.send();
    } else {
      xmlHttp.setRequestHeader("Content-Type", "application/json");
      xmlHttp.send(JSON.stringify(data));
    }
    xmlHttp.responseType = "json";
    xmlHttp.onreadystatechange = function(){
      if (xmlHttp.readyState !== 4) return;
      if (xmlHttp.status === 200) {
        resolve(xmlHttp.response);
      } else {
        reject(new Error(xmlHttp.statusText));
      }
    };
  });
  return promise;
};
```

### 28. 简述Promise

### 40. 优雅降级和渐进增强

**渐进增强（Progressive Enhancement）**：一开始就针对低版本浏览器进行构建页面，完成基本功能，然后再针对高级浏览器进行效果、交互追加功能。

**优雅降级（Graceful Degradation）**：一开始就构建站点的完整功能，然后针对浏览器测试和修复。

**区别**：优雅降级是从复杂的现状开始，试图减少用户体验的供给；渐进增强则是从基础版本开始，不断扩充。

### 41. JSONP的工作原理

JSONP是一种非正式传输协议，允许用户传递一个callback给服务端，服务端返回数据时会将这个callback参数作为函数名来包裹住JSON数据。

**为什么不是真正的Ajax**：
- ajax的核心是通过xmlHttpRequest获取非本页内容
- jsonp的核心是动态添加script标签调用服务器提供的js脚本
- jsonp只支持get请求，ajax支持get和post请求

### 56. 栈和堆的区别

1. **申请方式**：栈由系统自动分配，堆是人为申请开辟
2. **申请大小**：栈获得的空间较小，堆获得的空间较大
3. **申请效率**：栈速度较快，堆速度较慢
4. **存储内容**：栈在函数调用时存储调用语句地址和参数；堆在头部用字节存放堆的大小
5. **底层**：栈是连续的空间，堆是不连续的空间
6. **生长方向**：堆向上生长，栈向下生长
7. **管理方式**：栈由操作系统自动分配释放，堆由程序员控制

### 57. forEach、for...in、for...of的区别

1. **forEach**：遍历数组，不能使用break、continue和return语句
2. **for...in**：循环带有字符串key的对象，循环出的是key
3. **for...of**：循环出的是value，ES6新增，可以遍历数组对象

### 58. 对WebSocket的理解

WebSocket是一种网络传输协议，位于OSI模型的应用层。可在单个TCP连接上进行全双工通信。

**特点**：
- 较少的控制开销
- 更强的实时性
- 保持连接状态
- 更好的二进制支持
- 支持扩展
- 更好的压缩效果

### 59. JavaScript内存泄漏的几种情况

1. 意外使用全局变量
2. 定时器不清除
3. 闭包
4. 不清理DOM元素的引用
5. 监听事件未解除

### 60. 原生JS实现节流和防抖函数

**防抖**：
```javascript
const debounce = (func, wait = 50) => {
  let timer = 0;
  return function(...args) {
    if (timer) clearTimeout(timer);
    timer = setTimeout(() => {
      func.apply(this, args);
    }, wait);
  };
};
```

**节流**：
```javascript
export const throttle = function (fn, wait = 500) {
  let flg = true;
  return function () {
    if (!flg) return;
    flg = false;
    setTimeout(() => {
      fn.apply(this, arguments);
      flg = true;
    }, wait);
  };
};
```

## 二、HTTP/浏览器

### 1. HTTP和HTTPS

**基本概念**：
- **HTTP**：超文本传输协议，客户端和服务器端请求和应答的标准
- **HTTPS**：HTTP的安全版，HTTP下加入SSL层

**区别**：
1. HTTPS需要CA证书，费用较高
2. HTTP是明文传输，HTTPS是SSL加密传输
3. HTTP端口为80，HTTPS端口为443
4. HTTP连接简单无状态，HTTPS可进行加密传输、身份认证

**HTTPS工作原理**：
1. 客户使用https url访问服务器，要求建立SSL链接
2. 服务器返回证书（包含公钥）
3. 客户端和服务器协商SSL安全等级，建立会话密钥
4. 通过网站公钥加密会话密钥并传送
5. 服务器通过私钥解密出会话密钥
6. 服务器通过会话密钥加密与客户端通信

### 2. TCP和UDP的区别

1. TCP是面向连接的，UDP是无连接的
2. TCP提供可靠的服务（无差错、不丢失、不重复、按序到达）；UDP不保证可靠交付
3. TCP是面向字节流，UDP面向报文
4. TCP只能是1对1，UDP支持1对1、1对多
5. TCP首部20字节，UDP只有8字节
6. TCP是面向连接的可靠性传输，UDP是不可靠的

### 3. fetch发送2次请求的原因

fetch发送post请求时，第一次发送Options请求（状态码204），询问服务器是否支持修改的请求头；如果服务器支持，第二次才发送真正的请求。

### 4. Cookie、sessionStorage、localStorage的区别

**共同点**：都是保存在浏览器端，并且是同源的

**Cookie**：
- 数据始终在同源的http请求中携带
- 存储大小约4K
- 可以设置过期时间

**sessionStorage**：
- 仅在当前浏览器窗口关闭前有效
- 窗口关闭后消失

**localStorage**：
- 始终有效，窗口或浏览器关闭也一直保存
- 在同源窗口中共享

### 5. iframe是什么？有什么缺点？

**定义**：iframe元素会创建包含另一个文档的内联框架

**缺点**：
1. 会阻塞主页面的onload事件
2. 搜索引擎无法解读，不利于SEO
3. iframe和主页面共享连接池，影响性能

### 6. Cookie如何防范XSS攻击

XSS（跨站脚本攻击）是指攻击者在返回的HTML中嵌入javascript脚本。

**防范方法**：
- **httponly**：禁止javascript脚本访问cookie
- **secure**：仅在https请求时发送cookie

### 7. HTTP返回的状态码

- **1xx（信息）**：100 Continue、101 Switching Protocols
- **2xx（成功）**：200 OK、201 Created、204 No Content
- **3xx（重定向）**：301 Moved Permanently、302 Found、304 Not Modified
- **4xx（客户端错误）**：400 Bad Request、401 Unauthorized、403 Forbidden、404 Not Found
- **5xx（服务器错误）**：500 Internal Server Error、502 Bad Gateway、503 Service Unavailable

### 8. 强缓存、协商缓存

**强缓存**：直接从本地副本比对使用，不会发请求到服务器

**协商缓存**：会去服务器比对，若没改变才从本地缓存加载

**使用时机**：
- 资源未更新使用强缓存
- 资源已更新使用协商缓存获取最新资源

### 9. 前端优化

1. **降低请求量**：合并资源、减少HTTP请求数、minify/gzip压缩、webP、lazyLoad
2. **加快请求速度**：预解析DNS、减少域名数、并行加载、CDN分发
3. **缓存**：HTTP协议缓存、离线缓存manifest、localStorage
4. **渲染**：JS/CSS优化、加载顺序、服务端渲染

### 10. GET和POST的区别

1. GET参数通过url传递，POST放在request body中
2. GET请求有长度限制（实际是浏览器限制），POST没有
3. GET比POST更不安全，参数暴露在url中
4. GET请求浏览器主动cache，POST不会
5. GET产生一个TCP数据包，POST产生两个TCP数据包

### 11. 输入URL到页面加载显示完成发生了什么？

1. DNS解析
2. TCP连接
3. 发送HTTP请求
4. 服务器处理请求并返回HTTP报文
5. 浏览器解析渲染页面
6. 连接结束

### 12. CDN的优化原理

CDN加速的工作原理是将源站的资源缓存到全国各地的CDN节点上，用户访问时获取离自己最近的数据。

**好处**：
1. 提高安全性（隐藏源站）
2. 提升网站访问速度
3. 网站不容易挂机

### 13. 浏览器的内核

1. Mozilla Firefox的Gecko
2. Chrome的Blink（WebKit的分支）
3. Opera现为Blink
4. IE的内核Trident
5. Safari的内核WebKit

### 14. 浏览器是如何渲染页面的？

1. 解析HTML文件，创建DOM树
2. 解析CSS
3. 将CSS与DOM合并，构建渲染树（Render Tree）
4. 布局和绘制，重绘（repaint）和重排（reflow）

### 15. GET请求方式的长度限制

HTTP协议没有规定GET/POST的请求长度限制。GET请求参数的限制来源于浏览器或web服务器对URL长度的限制。

**各大浏览器限制**：
- IE：2083字节
- Chrome：8182字节
- Firefox：65536字符
- Safari：80000字符

### 16. 什么是同源策略？

同源策略是浏览器最核心也最基本的安全功能。同源指的是端口、域名、协议相同。

**阻止的操作**：
- 无法读取非同源的cookie、localstorage
- 无法接触非同源的dom
- 无法向非同源的地址发送ajax请求

### 17. 什么是跨域资源共享（CORS）？

CORS（Cross-Origin Resource Sharing）允许当前域的资源被其他域的脚本请求访问。

**使用方法**：服务端设置Access-Control-Allow-Origin

### 18. 什么是跨站请求伪造（CSRF）？

CSRF（Cross-site request forgery）是一种挟制用户在当前已登录的Web应用程序上执行非本意操作的攻击方法。

**预防方法**：
1. 提交验证码
2. Referer Check
3. Token验证
4. 在HTTP头中自定义属性并验证

### 19. 什么是跨站攻击（XSS）？

XSS（Cross Site Script Execution）是指攻击者利用网站程序对用户输入过滤不足，输入可以显示在页面上的HTML代码。

**预防方法**：
1. HttpOnly防止获取cookie
2. 输入检查（不要相信用户的所有输入）
3. 输出检查（转义或编码）

### 20. HTTP的几种请求方法用途

1. **GET**：发送请求来取得服务器上的某一资源
2. **POST**：向URL指定的资源提交数据
3. **PUT**：向服务器提交数据，指定资源在服务器上的位置
4. **HEAD**：只请求页面的首部
5. **DELETE**：删除服务器上的某资源
6. **OPTIONS**：获取当前URL所支持的方法
7. **TRACE**：激发远程应用层的请求消息回路
8. **CONNECT**：把请求连接转换到透明的TCP/IP通道

## 三、数据结构和算法

### 1. 什么是数组？

数组是由相同类型的元素的集合所组成的数据结构，分配一块连续内存来存储。

**特点**：相同类型、连续内存、固定长度。

### 2. JS中的数组是真正的"数组"么？

不是真正的数组，因为JS数组可以存放不同类型的值。

### 3. 什么是队列？

一种遵循**先进先出（FIFO）**原则的一组有序的项；队列在尾部添加新元素，并从头部移除元素。

### 4. 什么是链表？与数组的区别是？

存储有序的元素集合，但不同于数组，链表中的元素在内存中并不是连续放置的；每个元素由一个存储元素本身的节点和一个指向下一个元素的引用（指针/链接）组成。

**对比**：
- **数组**：连续且固定长度空间，查找高效，添加修改元素低效
- **链表**：不需要连续内存空间，大小可动态变化，查找低效，添加修改高效

### 5. 什么是栈？

一种遵从**先进后出（LIFO）**原则的有序集合；新添加的或待删除的元素都保存在栈的末尾，称作栈顶。

### 6. 什么是哈希及哈希冲突？

**Hash（散列/哈希）**：基本原理是把任意长度的输入，通过Hash算法变成固定长度的输出。

**哈希冲突**：不同的内容的hash值相同。

### 7. 二叉树有几种遍历方式？

三种：
1. 先序遍历
2. 中序遍历
3. 后序遍历

### 8. 简述冒泡排序

**基本思想**：冒泡排序类似于水中冒泡，较大的数沉下去，较小的数慢慢冒起来。较大的数慢慢往后排，较小的数慢慢往前排。
