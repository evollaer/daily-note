在人工智能时代，编写高质量的提示词已成为与AI高效协作的关键技能。本文旨在系统性地介绍如何构建有效的AI提示词，帮助开发者从基础的代码生成需求到复杂的工程任务都能获得精准、可靠的输出。我们将从核心原则出发，深入探讨APEC结构化方法，并通过丰富的实战案例展示优秀提示词的编写技巧，最终为您提供一套完整的提示词优化框架。
 



核心原则
1.
角色扮演（RolePlaying） ： 首先为AI定义一个明确的角色，这能引导它使用更专业的知识和模式。

不好 ： "写一个Node.js函数。"

更好 ： "你是一名资深Node.js开发工程师，擅长编写高效、可维护且符合ES6+规范的JavaScript代码。"

2.
任务清晰（ClearTask） ： 用精确的语言描述你要实现的功能。避免使用模糊、口语化的表达。

不好 ： "弄个东西来处理一下用户输入。"

更好 ： "编写一个函数，用于验证用户输入的电子邮件地址格式是否有效。"

3.
上下文丰富（RichContext） ： 提供必要的背景信息、输入输出示例、技术栈要求等。AI知道的越多，输出越精准。

不好 ： "连接数据库。"

更好 ： "使用Node.js的sqlite3包，创建一个到本地mydatabase.db文件的连接，并使用async/await处理可能发生的异常。"

4.
约束明确（ClearConstraints） ： 明确指定编程语言、版本、依赖库、代码风格、不允许使用的功能等。

不好 ： "排序一个数组。"

更好 ： "使用Node.js 18+，不依赖外部库，实现一个快速排序算法来对数字数组进行升序排序。"

5.
输出格式化（StructuredOutput） ： 明确要求你希望代码以何种形式呈现。是只要一个函数？还是一个完整的可运行脚本？是否需要注释？

不好 ： （无要求）

更好 ： "请只输出代码，不要任何解释。代码需要包含详细的JSDoc注释和使用示例。"

###黄金公式：APEC结构一个高效的代码提示词可以遵循 APEC* 结构：

A-Actor(角色) ： 你是谁？

P-Purpose(目标) ： 你要做什么？

E-Context(上下文) ： 背景信息是什么？（输入、输出、技术栈、示例等）

C-Constraints(约束) ： 有什么要求和限制？

*###实战案例对比####案例1：生成一个简单的函数 不友好的提示词：>写个函数加数字。 更友好的提示词（应用APEC结构）：> 角色（Actor）：你是一名Node.js专家。 目标（Purpose）：编写一个函数来计算两个数字的和。 上下文（Context）：函数需要处理整数和浮点数。 约束（Constraints）：>>1.函数名称为addNumbers。>2.使用ES6+语法和JSDoc注释。>3.代码符合JavaScript最佳实践。>4.只需输出函数代码本身，不需要示例调用。 AI生成的代码可能如下：```javascript/

计算两个数字的和

@param {number} a - 第一个数字

@param {number} b - 第二个数字

@returns {number} 两个数字的和
*/
function addNumbers(a, b) {
   return a + b;
}
​
// 或者使用箭头函数
const addNumbers = (a, b) => a + b;

 
#### 案例2：完成一个复杂任务（爬虫）

**不友好的提示词：** 

> 帮我爬一下某个网站的产品价格。

**更友好的提示词：** 

> **角色** ： 你是一名专业的Node.js爬虫工程师。 **目标** ： 编写一个脚本，从 `https://example.com/products` 抓取所有产品的名称和价格。 **上下文** ：
>
> 1. 产品列表在HTML中，每个产品都有一个CSS类名为 `.product`。
> 2. 产品名称在 `.product-name` 标签内。
> 3. 产品价格在 `.product-price` 标签内。
> 4. 请使用 `axios` 和 `cheerio` 库。 **约束** ：
> 5. 添加必要的异常处理（如网络请求失败、解析失败）。
> 6. 设置合理的请求头（User-Agent），模仿浏览器行为。
> 7. 代码结构清晰，有注释，使用async/await语法。
> 8. 最终将数据以JSON列表的形式输出到控制台，格式如：`[{"name": "Product A", "price": "$29.99"}, ...]`

**这样，AI就会生成一个结构完整、考虑周全的脚本，而不是一个半成品。** 

***###高级技巧1.** 迭代式交互**： 不要期望一次成功。可以把复杂任务拆解。

   * **第一轮** ： "帮我设计一个MongoDB数据库集合结构，用于存储用户博客文章。"
   * **第二轮** ： "基于上面的集合结构，写一个Node.js类，用Mongoose ODM 来定义这些模型。"
   * **第三轮** ： "现在，为这个模型类编写增删改查（CRUD）的方法。"
2. **提供示例（Few-ShotLearning）** ： 如果你有特定的代码风格或模式，可以直接给AI一个例子。

   > 请按照以下格式编写一个类似的函数：
   >
   > javascript
   >
   > ```
   > /**
   >  * 处理数据函数
   >  * @param {Array} inputData - 输入数据数组
   >  * @returns {Object} 处理结果对象
   >  */
   > async function processData(inputData) {
   >     // ... 异步代码逻辑 ...
   >     return result;
   > }
   > ```
   >
   > 现在，请编写一个 `calculateStatistics` 函数...
3. **要求优化和解释** ： 生成代码后，你可以继续提问。

   * "这段代码的时间复杂度是多少？"
   * "有没有性能更好的写法？"
   * "请为这段代码添加错误处理。"
4. **指定编码方式** ：

   * "请使用Promise链实现。"
   * "请使用async/await语法来简化异步操作。"
   * "请使用数组的map/filter/reduce方法来处理数据。"

### 总结： checklist

在写出提示词后，问自己几个问题：

* **角色** ： AI是否清楚自己的专业领域？
* **目标** ： 任务描述是否**具体、无歧义** ？
* **输入/输出** ： 是否明确了输入数据的格式和期望输出的格式？
* **技术栈** ： 是否指定了编程语言、版本、关键库？
* **约束** ： 是否说明了代码风格、禁止使用的功能、异常处理等？
* **格式** ： 是否说明了需要代码还是完整脚本？是否需要注释？

通过有意识地结构化你的提示词，AI（如ChatGPT、Claude、DeepSeek Coder等）将会从一个“有点聪明的助手”蜕变成一个“高度专业、输出精准的编程伙伴”。
